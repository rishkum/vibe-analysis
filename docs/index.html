Preface

Welcome to Vibe Analysis.

This book is about a simple but consequential idea: the way we write code for data analysis is changing fundamentally, and the skills required to do it well are changing with it.

“Vibe coding”; the practice of using natural language to instruct AI models to generate executable code; has captured the imagination of the software world. Tools like GitHub Copilot can translate a plain English comment into a working function in seconds. For building applications, this is a revolution. For extracting analytical insights, it is a loaded gun.

The difference matters. A vibe-coded web application with a minor UI bug still works. A vibe-coded data pipeline with a silent aggregation error produces numbers that look perfectly plausible but are fundamentally wrong. Decisions get made, reports get published, and nobody realises the foundation was flawed until it is too late.

Vibe Analysis is the methodology this book proposes to bridge that gap. It takes the speed and accessibility of vibe coding and layers on the rigour that analytical work demands: structured code reviews, statistical validation, explicit assumption-testing, and human ownership of every result.

If you spend more time trying to remember the exact arguments of dplyr::pivot_longer() or the aesthetics of a ggplot2 chart than thinking about the insights your data contains, this book is for you. If you manage a team of analysts who have started using AI tools and you want to know what questions to ask, this book is also for you.

Vibe Coding vs. Vibe Analysis

To ensure clarity throughout this book, we distinguish two related but critically different concepts:

  • Vibe Coding: The general mechanism of writing software using natural language prompts. It prioritises speed, compilation, and whether the code runs.
  • Vibe Analysis: The rigorous methodology of applying that mechanism to data science. It introduces statistical rigour, code reviews drawn from the Software Development Lifecycle (SDLC), and hypothesis testing to ensure the rapid code generation does not produce hallucinated insights.

Why Vibe Analysis?

In traditional data analysis, the bottleneck is often the translation of a statistical idea into programming syntax. You have a question (“How do sales compare across regions over the last three quarters?”), but you must spend time writing boilerplate code to group, summarise, and plot the data.

With AI assistants; specifically GitHub Copilot, the primary tool explored in this book; you can write the question in plain English as a comment, and the assistant will synthesise the code. Your role as an analyst shifts:

  1. From typist to architect: You define the analytical strategy and the questions worth asking.
  2. From debugger to auditor: You review and validate the logic the AI proposes, rather than writing it from scratch.
  3. From memoriser to thinker: You focus on research design and statistical validity rather than syntax recall.

This shift is powerful, but only if you maintain the discipline to verify what the AI produces. The chapters that follow will show you how.

Best Practices for the Vibe Workflow

To get the most out of GitHub Copilot in your data analysis workflow, adopt these core practices from the outset:

  • Plan, Act, Review: Slice your analytical tasks into smaller, manageable steps. Do not ask for a full dashboard in a single prompt. Ask for data import, then cleaning, then visualisation; reviewing the output at each stage.
  • Context is King: The AI needs to see your data structure. Print the head() or str() of your data frame so Copilot understands the variable names and types before generating transformations.
  • Human-in-the-Loop: Always verify intermediate steps. A model might confidently generate code that looks correct but performs an aggregation incorrectly; for example, using mean() when you needed median().
  • Create Persistent Context: Document your data assumptions in a PROJECT.md file or at the top of your script. This gives Copilot a consistent reference point and keeps it aligned with your goals across sessions.
